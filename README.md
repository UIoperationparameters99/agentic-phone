# agentic-phone

> BYOK AI agentic workspace for Android — a faithful clone of [z.ai agentic mode](https://chat.z.ai), built for the phone.

[![License: Apache-2.0](https://img.shields.io/badge/License-Apache_2.0-blue.svg)](https://opensource.org/licenses/Apache-2.0)

## What this is

A mobile-first AI agent app where the AI has its **own Linux computer in the cloud** — full bash, file system, web access, code execution, and a workspace it controls. You bring your own LLM API keys (OpenAI, Anthropic, Google, OpenRouter, Z.AI) and your own cloud sandbox (Daytona — free, no credit card). The app is installable as an APK on Android.

Think: **Cursor + Claude Code + z.ai agentic mode, on your phone, BYOK, smooth.**

## Architecture (one-screen summary)

```
┌────────────────────────────────────────────────────────────┐
│  MOBILE  (Next.js 16 + Capacitor 8 → APK)                  │
│  ─ BYOK: LLM keys + Daytona key in Android Keystore        │
│  ─ Capacitor HTTP plugin → Daytona REST (no CORS, native)  │
│  ─ WebSocket → sandbox preview URL (event stream)          │
│  ─ Chat UI + tool-call cards + file browser + todo panel   │
└────────────────────────────────────────────────────────────┘
              │                                    │
              │  1. POST /sandbox (Capacitor HTTP) │  2. WS connect
              ▼                                    ▼
┌────────────────────────────────────────────────────────────┐
│  CLOUD LINUX COMPUTER  (Daytona Sandbox, BYOS)             │
│  ─ @cline/sdk agent loop (Apache-2.0, npm dep)             │
│  ─ Tools: bash, read, write, edit, grep, glob, ls, todo    │
│  ─ Skills: SKILL.md loader (mirrors z.ai's pattern)        │
│  ─ Calls LLM with user's key (env var, wiped on destroy)   │
│  ─ Persists via snapshot + git                             │
└────────────────────────────────────────────────────────────┘
              │
              │  3. HTTPS to LLM provider
              ▼
        [OpenAI / Anthropic / Google / OpenRouter / Z.AI]
```

**BYOS — Bring Your Own Sandbox.** Same model as BYOK. Each user signs up for their own Daytona account (free $100 credit, no credit card required, email verification only), pastes their Daytona API key into the app alongside their LLM keys. You carry $0 infrastructure cost — every user burns their own Daytona credit.

**BYOK transport — Option B.** LLM keys are passed from the mobile Keystore to the sandbox as environment variables at spawn time. The agent (running inside the sandbox) calls the LLM directly — no double-hop, mobile can disconnect mid-agent-run (true background autonomy). The sandbox is per-user, isolated, and ephemeral; keys are wiped when the sandbox is destroyed.

## Repo layout

```
agentic-phone/
├── apps/
│   ├── mobile/          Next.js 16 + Capacitor 8 → APK
│   │   ├── app/         Chat UI, tool-call cards, file browser, skill picker, BYOK config
│   │   ├── components/  shadcn/ui-based components
│   │   ├── lib/         byok storage, daytona client, ws client, event renderer
│   │   └── android/     Capacitor Android shell (generated by `cap add android`)
│   └── sidecar/         Bun + @cline/sdk, runs INSIDE the Daytona sandbox
│       ├── src/
│       │   ├── index.ts           Entry — starts WS server
│       │   ├── agent.ts           @cline/sdk AgentRuntime setup
│       │   ├── tools/             bash, read, write, edit, grep, glob, ls, todo
│       │   ├── skills/            SKILL.md loader (z.ai pattern)
│       │   └── transport/         WS server, event envelope
│       └── package.json
├── packages/
│   └── shared-types/    Event protocol types shared by mobile + sidecar
└── docs/
    ├── architecture.md  Full architecture doc
    ├── byok-transport.md BYOK Option A vs B decision
    └── sandbox-setup.md Daytona setup guide for end users
```

## Development

```bash
# Install deps
pnpm install

# Run mobile + sidecar in parallel (dev)
pnpm dev

# Build APK
pnpm mobile:apk
```
